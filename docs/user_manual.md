## 1. 系統需求 / System Requirements

| 項目 | 需求 |
|------|------|
| 桌機瀏覽器 | Google Chrome ≥ 89 或 Microsoft Edge ≥ 89 |
| 行動裝置 | Android Chrome（需 WebUSB 支援） |
| 作業系統 | Windows / macOS / Linux / Android |
| 硬體介面（桌機） | USB Type-A，支援 FTDI FT232R 晶片（VID 0403h / PID 6001h） |
| 硬體介面（Android） | USB OTG 線（Micro-USB 或 USB-C 轉 Type-A） |

> **注意**：Firefox、Safari 不支援 Web Serial API 和 WebUSB，無法使用本應用程式。
> **Note**: Firefox and Safari do not support Web Serial API or WebUSB.

## 2. 首頁：裝置連接 / Home: Device Connection

首頁是應用程式的起始點，顯示裝置連線狀態與基本資訊。

### 狀態指示燈

| 顏色 | 狀態 |
|------|------|
| 綠色（發光） | 已連接 |
| 藍色（閃爍） | 連接中… |
| 灰色 | 未連接 |
| 紅色 | 連接錯誤 |

### 連接後顯示資訊

- **裝置序號**：FT232R 晶片的序號或產品名稱
- **採樣率**：固定 1000 Hz
- **封包速率**：每秒收到的資料封包數（正常應為約 10 pkt/s）
- **電池**：以彩色電量條顯示，> 50% 綠色，> 20% 黃色，≤ 20% 紅色

### 操作步驟（標準桌機流程）

1. 點擊「**連接裝置**」按鈕，開啟連線視窗
2. 在連線視窗選擇序列埠後點擊「**連線**」
3. 連線成功後，狀態燈轉為綠色，頁面顯示裝置資訊
4. 點擊「**斷開連接**」可隨時中止連線

## 3. 連線視窗 / Connect Modal

點擊「連接裝置」後開啟，分為**桌機模式**與 **Android 模式**。

### 3.1 桌機模式（Web Serial API）

**已授權的 FTDI 裝置列表**

- 首次使用時列表為空；點擊「**授權新裝置（首次使用）**」讓瀏覽器顯示 COM port 選擇器
- 授權過的裝置會以序號列出，每台裝置旁顯示：
  - **配對狀態**：是否已關聯 COM port（同個序號可能對應不同 COM 號）
  - **已配對 COM port** / **尚未配對 COM port**
- 點擊裝置項目可選取，點擊「**配對 COM Port**」可重新讓瀏覽器選擇 COM port

**連線按鈕行為**

- 若選取裝置已配對 COM port → 直接連線
- 若有多個已授權序列埠 → 開啟瀏覽器 picker 再選一次
- 若選取裝置尚未配對 → 提示先配對

**COM port 不符警告**

若選取的裝置序號與 COM port 的序號不一致，畫面會顯示橘色警告：「⚠️ COM port 不符！」請選擇正確的 COM port。

**清除授權**

點擊「**清除所有裝置授權**」可移除瀏覽器所有已記憶的序列埠授權記錄。

### 3.2 Android 模式（WebUSB）

當瀏覽器不支援 Web Serial API（如 Android Chrome）時，畫面自動切換至 Android USB 直連區塊。

1. 以 USB OTG 線將 EEG 裝置連接至手機
2. 點擊「**選擇 USB 裝置**」，Chrome 顯示 USB 裝置選擇器
3. 選擇 FT232R 裝置後，App 自動完成以下 FTDI 初始化：
   - 晶片重置
   - 設定 1 Mbaud 速率
   - 設定 8N1 格式（8 位元，無同位，1 停止位）
   - 關閉流量控制
   - 設定延遲計時器 1 ms
   - 清除 RX / TX 緩衝區
4. 初始化完成後自動切換至「已連接」狀態

> 若顯示「此瀏覽器不支援 WebUSB，無法直連裝置」，請確認使用 Android Chrome 且版本足夠新。

## 4. 阻抗頁面 / Impedance Tab

用於連接前檢查電極接觸品質。

> **互斥限制**：阻抗量測進行中無法錄製資料；錄製中也無法開始阻抗量測。

### 電極位置圖

以 SVG 頭部輪廓呈現 8 個電極的 10-20 位置：

```
        Fp1  Fp2
     T7    Fz    T8
           Pz
        O1   O2
```

### 阻抗品質等級

| 顏色 | 等級 | 阻抗範圍 |
|------|------|---------|
| 綠色 | 優秀 Excellent | < 150 kΩ |
| 淺綠 | 良好 Good | 150–300 kΩ |
| 橘色 | 尚可 Poor | 300–600 kΩ |
| 紅色 | 不良 Bad | ≥ 600 kΩ |
| 灰色 | 未量測 | — |

若 AC 訊號振幅 < 0.5 µV 或阻抗 < 10 kΩ，顯示「**N/A**」（未偵測到訊號，可能電極未接觸）。

### 操作步驟

1. 確認已連接裝置
2. 點擊「**開始量測**」，裝置進入阻抗量測模式（持續更新）
3. 調整電極直到所有通道呈現綠色或淺綠
4. 點擊「**停止量測**」結束，回到正常 EEG 串流模式

## 5. 訊號頁面 / Signal Tab

即時顯示 8 通道 EEG 波形，使用 WebGL 加速渲染。

### 介面說明

- **縱軸**：每個通道各一條波形，從上到下依序為 Fp1、Fp2、T7、T8、O1、O2、Fz、Pz
- **橫軸**：時間（滑動視窗），可調整顯示時長
- **白色游標線**：指示目前資料寫入位置

### 控制選項

| 控制 | 說明 |
|------|------|
| **時間 (Time)** | 顯示時間視窗長度（秒） |
| **幅度 (Scale)** | Y 軸縮放比例（µV/格） |
| **自動 (Auto)** | 自動調整幅度至訊號範圍 |
| **帶通 (Bandpass)** | 啟用/停用帶通濾波器 |
| **高通 HP (Hz)** | 高通截止頻率（預設 1 Hz） |
| **低通 LP (Hz)** | 低通截止頻率（預設 45 Hz） |
| **陷波 (Notch)** | 關 / 50 Hz / 60 Hz（電源線噪音消除） |

### 事件標記

- 按 **Space** 鍵或 **M** 鍵可在目前時間點新增事件標記
- 標記以垂直彩色線顯示在波形上，標記旁顯示時間戳
- 點擊「**清除全部**」可刪除所有標記
- 標記列表顯示在頁面右側

### 濾波器預設值

| 參數 | 預設值 |
|------|------|
| 帶通 | 啟用 |
| 高通 | 1 Hz |
| 低通 | 45 Hz |
| 陷波 | 60 Hz |

## 6. 頻譜頁面 / FFT Tab

即時顯示功率頻譜密度（PSD），以 Hann 視窗 FFT 計算。

### 控制選項

- **最大頻率 (Max Freq)**：X 軸上限（Hz），可根據需要調整顯示範圍

### 說明

- 縱軸為功率（µV²/Hz）

## 7. 記錄頁面 / Record Tab

用於記錄 EEG 資料及生成分析報告。

### 7.1 受試者資訊

錄製開始前輸入以下資料（供 CSV 檔頭及報告使用）：

| 欄位 | 說明 |
|------|------|
| **受試者編號** | 受試者 ID（可為英數字混合） |
| **姓名** | 受試者姓名 |
| **出生年月日** | YYYY-MM-DD 格式，用於 T-score 年齡校正 |
| **性別** | 男 / 女 / 其他 |
| **備注** | 任意文字備注 |

### 7.2 訊號品質監控

| 控制 | 說明 |
|------|------|
| **啟用 / 關閉** | 開啟或關閉品質監控功能 |
| **靈敏度** | 寬鬆 / 嚴格（調整雜訊判斷閾值） |
| **目標時長** | 30 / 60 / 90 / 120 / 150 / 180 / 300 秒，或手動停止 |

品質監控會排除含有大幅雜訊的時段：
- 達到目標時長後自動停止並顯示「**已達目標時長，自動停止**」
- **有效時間 (Good Time)**：顯示已累積的有效錄製時間（MM:SS）
- **品質百分比 (Quality %)**：有效時間佔總錄製時間的比例

### 7.3 錄製控制

| 按鈕 | 說明 |
|------|------|
| **開始錄製** | 開始將 EEG 資料寫入記憶體緩衝區 |
| **停止** | 停止錄製，不自動下載 |
| **停止並下載** | 停止錄製並下載原始 CSV 檔案 |
| **停止並生成報告** | 停止錄製並執行 CCA + EEG 指標分析，產生 PDF 報告 |

> **注意**：生成報告需要至少 **90 秒**的有效錄製資料。

### 7.4 CSV 檔案格式

下載的 CSV 檔案格式如下：
- **標頭列**：受試者資訊（ID、姓名、DOB、性別、備注）、裝置 ID、濾波設定
- **資料列**：時間戳（ms）、序號、8 個通道的 µV 值

檔名格式：`eeg_{ID}_{YYYY-MM-DDTHH-MM-SS}.csv`

### 7.5 事件標記記錄

- 在錄製中按 **Space** / **M** 可新增事件標記
- 也可點擊「**新增標記**」按鈕
- 標記記錄顯示於頁面下方，包含時間戳（HH:MM:SS.mmm）
- 「**清除全部**」可刪除全部標記

### 7.6 讀取 CSV 生成報告

若已有先前錄製的 CSV 檔案，可直接載入分析：

1. 輸入**出生年月日**（用於 T-score 年齡校正）
2. 點擊「**讀取 CSV 生成報告**」，選擇 CSV 檔案
3. 系統解析後顯示樣本數與時長
4. 自動執行 CCA 去雜訊 + 7 項 EEG 指標計算
5. 完成後自動下載 PDF 報告

> CSV 檔案需至少 90 秒資料才可生成報告。

## 9. 報告生成 / Report Generation

### 9.1 CCA 去雜訊

在生成報告前，系統自動使用**典型相關分析（CCA）**去除 EEG 中的肌電（EMG）和眼動（EOG）雜訊：

1. **分段處理**：將 EEG 切為 2 秒（2002 個樣本）的 epoch
2. **壞 epoch 偵測**：排除標準差或峰對峰值超過 IQR 閾值的 epoch，並排除振幅 > 195 µV 的 epoch
3. **CCA 分解**：對剩餘 epoch 執行 CCA（Cholesky 白化 + Jacobi 特徵值分解）
4. **成分分類**：
   - **肌電成分**：自相關係數 < 0.9（高頻特性）
   - **眼動成分**：前 2 個成分中，Delta 頻段功率佔比 > 65% 且 peak/RMS > 6.0
5. **重建**：移除上述成分後，從剩餘成分重建乾淨 EEG

### 9.2 七項 EEG 指標

| 指標 | 全名 | 通道 | 頻段 |
|------|------|------|------|
| **TBR** | Theta-Beta Ratio | Fz, Pz | θ (4–8 Hz) / β (13–30 Hz) |
| **APR** | Alpha Power Ratio | T7, T8, Fz, Pz | α (8–13 Hz) / total |
| **FAA** | Frontal Alpha Asymmetry | F3, F4 → Fz/Fp 校正 | α (8–13 Hz) |
| **PAF** | Peak Alpha Frequency | O1, O2 | α (7–14 Hz) 重心 |
| **RSA** | Relative Spectral Asymmetry | O1, O2 | α1 (8–10 Hz) / α2 (10–13 Hz) |
| **COH** | Coherence | Fp1↔Fp2 等 6 對 × 5 頻段 | δ θ α β γ |
| **EnTP** | Entropic Power | Fp1, Fp2, T7, T8, O1, O2 | 排列熵（order=3） |

所有指標均以 **Hann 視窗 FFT 功率頻譜 + Simpson 積分**計算，並使用 IQR 法（乘數 1.2）去除 epoch 異常值，最後轉換為 **T-score（均值 50，標準差 10）**。

### 9.3 PDF 報告內容

- 受試者基本資訊
- 去雜訊統計（移除成分數、保留 epoch 數）
- 7 項指標的 T-score 長條圖與數值
- 錄製時間、通道資訊

## 10. Android 手機操作 / Android Operation

### 需求

- Android 手機（Android 6.0+）
- Google Chrome for Android（最新版）
- USB OTG 轉接線（Micro-USB 或 USB-C 轉 Type-A）

### 步驟

1. 用 USB OTG 線連接 EEG 裝置與手機
2. 開啟 Chrome，瀏覽至 sgimacog EEG App 網址
3. 點擊「**首頁 → 連接裝置**」
4. 畫面顯示「**Android USB 直連**」區塊（因為無 Web Serial API）
5. 點擊「**選擇 USB 裝置**」，Chrome 顯示 USB 裝置選擇器
6. 選擇 FT232R 裝置，點允許授權
7. App 自動完成 FTDI 協定初始化（約 1–2 秒）
8. 連線成功，可正常使用所有功能

### Android 限制

- 不支援「已授權裝置列表」（Web Serial API 功能，桌機限定）
- 每次開啟 App 都需重新授權一次
- 部分舊款 Android 裝置可能不支援 WebUSB

## 11. 多裝置同時連線 / Multi-Device

每個瀏覽器分頁只能連接一台裝置。若需同時記錄多台：

1. 在不同瀏覽器**分頁**中各自開啟本應用程式
2. 每個分頁分別連線不同的 COM port / USB 裝置
3. 各分頁獨立運作，互不干擾

## 12. 常見問題 / Troubleshooting

### 連線問題

| 問題 | 解決方式 |
|------|---------|
| 找不到 COM port | 確認 FTDI 驅動程式已安裝（Windows：Device Manager > Ports） |
| 授權後仍無裝置 | 點「重新掃描」或重新拔插 USB 後再試 |
| 連線後立即斷線 | 確認 baud rate 正確（1,000,000 baud），不要使用 USB Hub |
| Android 無法選擇裝置 | 確認使用 USB OTG 線，且 USB 裝置已通電 |

### 訊號問題

| 問題 | 解決方式 |
|------|---------|
| 阻抗顯示 N/A | 電極可能未接觸皮膚，或導電膠不足 |
| 波形全是直線 | 確認電極接觸良好，阻抗 < 600 kΩ |
| 波形噪音很大 | 啟用陷波（50/60 Hz），縮小帶通範圍，檢查接地 |
| 封包速率異常低 | 檢查 USB 連線，確認無其他應用程式佔用同一 COM port |

### 報告問題

| 問題 | 解決方式 |
|------|---------|
| 「資料不足 90 秒」 | 需至少 90 秒的錄製資料才能分析 |
| CSV 格式錯誤 | 只支援本 App 產生的 CSV 格式 |
| 報告生成失敗 | 確認訊號品質良好，有效 epoch 數量足夠 |

### 瀏覽器相容性

| 瀏覽器 | 桌機 | Android |
|--------|------|---------|
| Chrome 89+ | ✅ | ✅（WebUSB） |
| Edge 89+ | ✅ | ✅（WebUSB） |
| Firefox | ❌ | ❌ |
| Safari | ❌ | ❌ |
